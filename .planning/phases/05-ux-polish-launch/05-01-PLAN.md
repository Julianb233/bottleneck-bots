## Audit Checklist Per Page

For each page, check:

### Rendering
- [ ] Page loads without white screen
- [ ] No JavaScript console errors
- [ ] No missing component errors
- [ ] All text is visible and readable
- [ ] Images/icons load correctly

### Interactive Elements
- [ ] All buttons are clickable and trigger expected action
- [ ] All form inputs accept text
- [ ] Dropdowns open and options are selectable
- [ ] Modals/dialogs open and close correctly
- [ ] Tabs switch content correctly

### Navigation
- [ ] Sidebar navigation links work
- [ ] Breadcrumbs link to correct pages
- [ ] Back button works as expected
- [ ] URL updates correctly on navigation

### Data Loading
- [ ] Loading states shown while data fetches
- [ ] Data populates correctly after load
- [ ] Empty states shown when no data
- [ ] Pagination works if present
- [ ] Error states shown on API failure

### Responsive Design
- [ ] Desktop (1920x1080) -- layout correct
- [ ] Tablet (768x1024) -- responsive adjustments
- [ ] Mobile (375x667) -- mobile-friendly layout

### Accessibility
- [ ] Form labels associated with inputs
- [ ] Color contrast sufficient
- [ ] Keyboard navigation works (tab order)

## Issue Classification

| Severity | Definition | Action |
|----------|-----------|--------|
| **HIGH** | Blocks core functionality (white screen, broken auth, can't create task) | Fix immediately in this plan |
| **MEDIUM** | Functionality works but UX is poor (layout broken, confusing UI) | Fix if time permits |
| **LOW** | Cosmetic issues (alignment, spacing, icon size) | Log for future fix |

---

## Tasks

### 1. Automated Page Crawl

Use Stagehand to visit every page:
- Navigate to each URL
- Wait for page load (no spinners)
- Take screenshot (desktop + mobile viewport)
- Check console for errors (`page.on('console', ...)` for error messages)
- Check network for failed requests (4xx/5xx)

### 2. Interactive Testing

For pages with forms:
- **Login**: Enter test email + password, submit, verify redirect
- **Signup**: Fill registration form, submit (use test+timestamp@example.com)
- **Agent Dashboard**: Type a task, submit (verify execution starts or appropriate error)
- **Workflow Builder**: Drag a node, connect two nodes, save
- **Training**: Upload a test document
- **Scheduled Tasks**: Create a new schedule, verify it appears
- **Settings**: Change a setting, verify it saves
- **Lead Upload**: Upload a test CSV

### 3. Generate Audit Report

**File**: `docs/ux-audit-report.md`

Format:
```markdown

Date: 2026-03-20
Tester: Agent (Stagehand automated)
Environment: Production (Vercel URL)

- HIGH issues: X
- MEDIUM issues: X

- **URL**: /path
- **Steps to reproduce**: ...
- **Expected**: ...
- **Actual**: ...
- **Screenshot**: [link]
- **Fix**: [description of fix applied]

```

### 4. Fix All HIGH Issues

For each HIGH issue found:
- Identify the root cause in the source file
- Apply the fix
- Re-test with Stagehand to verify fix
- Note the fix in the audit report

Common HIGH issues to watch for:
- Missing environment variables causing API errors
- Broken imports from refactoring
- React 19 / Tailwind 4 compatibility issues
- Authentication redirects not working
- tRPC procedures returning errors
