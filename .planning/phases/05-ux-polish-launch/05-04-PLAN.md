### 9. Final Fix Pass

For any issues found in tests 1-18:
- Categorize as CRITICAL (blocks launch) or MINOR (can launch with)
- Fix all CRITICAL issues immediately
- Log MINOR issues in `docs/known-issues.md` with planned fix dates

### 10. Launch Steps

Once all CRITICAL issues resolved:

1. **Final build**: Push to main, verify Vercel deployment succeeds
2. **Domain**: Ensure DNS is pointed to Vercel (if custom domain)
3. **Stripe**: Switch webhook URL from test to production
4. **GHL**: Update OAuth redirect URI to production domain
5. **Sentry**: Configure release tracking (`sentry-cli releases new`)
6. **Monitoring**: Set up Vercel uptime monitoring or external pingdom/betteruptime
7. **Announcement**: Draft launch message for:
   - Social media (Twitter/X, LinkedIn)
   - Email list (if exists)
   - Relevant communities (GHL community, agency forums)
8. **Monitor first hour**: Watch Sentry, Vercel logs, Stripe dashboard for issues

## Risks

- Production environment differences from dev (env vars, network, DNS)
- Stripe live mode vs test mode confusion (ensure using correct keys)
- GHL OAuth redirect URI must exactly match registered app
- First real users may hit edge cases not covered in smoke tests
- Vercel cold starts may cause slow first-request times
- Redis connection issues on serverless (use connection pooling)
