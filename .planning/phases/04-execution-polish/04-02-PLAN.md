**Day**: 4 (Thu 3/19)

## Tasks

### 1. Enhance Execution History Sidebar

**File**: `client/src/components/agent/ExecutionHistory.tsx` (enhance)

Add:
- **Filter controls** at top: status dropdown (all/success/failed/cancelled), date range picker
- **Search** by task description text
- **Sort** toggle: newest first / oldest first
- **Grouping**: group by date (Today, Yesterday, This Week, Older)
- **Compact vs detailed** view toggle
- Each item shows: task description (truncated), status badge, duration, credit cost, timestamp
- Click loads full detail in main panel

### 2. Create Execution Detail Page/Panel

**File**: `client/src/components/agent/ExecutionDetailView.tsx` (new)

Full execution detail view that replaces the main panel content when viewing past execution:

**Header**:
- Task description (full)
- Status badge (success/failed/cancelled)
- Duration, credits used, timestamp
- Action buttons: "Retry", "View Recording", "Export PDF", "Delete"

**Tabs**:
1. **Timeline**: Step-by-step timeline of all actions taken
   - Each step: action type icon, description, duration, status
   - Expand step to see: screenshot (if captured), LLM reasoning, selector used, result
   - Failed steps highlighted in red with error message
   - Reuse `StepTimeline.tsx` and `PhaseProgressTimeline.tsx` components

2. **Reasoning**: Full LLM reasoning chain
   - Show each thinking step in order
   - Planning phase highlighted separately
   - Tool calls with inputs/outputs
   - Reuse `ReasoningChain.tsx` and `AgentThinkingViewer.tsx`

3. **Browser Recording**: Embedded session replay
   - If BrowserBase recording URL available: show embedded replay using `SessionRecordingViewer.tsx`
   - Playback controls: play, pause, speed (1x, 2x, 4x), scrub bar
   - rrweb integration (already in dependencies per ROADMAP.md)
   - If no recording: show "Recording not available" with explanation

4. **Errors** (only if execution failed):
   - Error type classification (from `failureRecovery.service.ts` error types)
   - Full error message and stack trace
   - Recovery attempts made
   - Suggested fixes
   - Reuse `ExecutionErrorDisplay.tsx`

5. **Output**: Execution result/output data
   - JSON viewer for structured output
   - Extracted data display
   - Generated files/reports

### 3. Wire Execution Loading from Backend

Ensure `agentStore.loadExecution(executionId)` fetches complete data:
- Task description, status, timestamps, duration
- All thinking steps with reasoning
- Browser session ID and recording URL
- Screenshots captured during execution
- Error details if failed
- Output/result data
- Credit cost

If the existing tRPC procedure `agent.getExecution` doesn't return all this, enhance it.

### 4. Export Execution Report

Integrate with existing `pdf-report.service.ts`:
- Add tRPC procedure `agent.exportExecutionReport` -- input: `{ executionId, format: 'pdf' | 'json' }`
- PDF includes: task summary, step timeline, screenshots, errors, output
- JSON includes: raw execution data for programmatic use
- Download triggers file save dialog

### 5. Retry Failed Execution

Add "Retry" functionality:
- Button on failed execution detail view
- Creates a new execution with the same task description
- Optionally allows editing the task before retry
- Links the retry to the original execution for tracking

1. View execution history -> shows grouped list with filters
2. Filter by "failed" -> only failed executions shown
3. Search "contact" -> finds executions with "contact" in task description
4. Click an execution -> detail view loads with all tabs

6. Reasoning tab shows full LLM thinking chain
7. Browser Recording tab plays back the session (if recording exists)

## Risks

- rrweb recordings may be large and slow to load for long sessions
- BrowserBase recording URLs may expire after a certain time
- Historical executions may not have complete step data (if format changed)
- PDF generation for long executions could be slow
