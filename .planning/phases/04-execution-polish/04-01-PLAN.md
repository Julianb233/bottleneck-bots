## Tasks

### 1. Enhance Right Panel Preview in AgentDashboard

**File**: `client/src/pages/AgentDashboard.tsx`

Replace the simple iframe in the "preview" TabsContent with an enhanced component:

```tsx
<TabsContent value="preview" className="flex-1 m-0 overflow-hidden">
  <EnhancedLivePreview
    session={activeBrowserSession}
    isExecuting={isExecuting}
    sseConnected={sseConnected}
    executionId={currentExecution?.id}
  />
</TabsContent>
```

### 2. Create EnhancedLivePreview Component

**File**: `client/src/components/agent/EnhancedLivePreview.tsx`

Features:
- **Status bar** (top): Live/recording indicator, SSE connection status, current URL display, elapsed time
- **Control bar** (below status):
  - Screenshot capture button (calls tRPC `browser.screenshot` or BrowserBase API)
  - Zoom controls (50%, 75%, 100%, 125%, fit-to-panel) -- CSS transform on iframe container
  - "Open in New Tab" button (existing)
  - "View Recording" link (opens BrowserBase recording URL when session completes)
  - Pause/Resume toggle (calls `agent.pauseExecution` / `agent.resumeExecution`)
- **Main view**: iframe with zoom applied via `transform: scale()` + `transform-origin: top left`
- **Action overlay** (bottom of iframe): Semi-transparent bar showing current agent action
  - "Clicking: Submit button" / "Typing: search field" / "Navigating to: /contacts"
  - Fed by SSE thinking steps from `useAgentSSE`
- **Empty state**: Show when no active session (existing, but improved with instructions)

### 3. Screenshot Capture Integration

Add to agent tRPC router or browser router:
- `browser.captureScreenshot` mutation: input `{ sessionId }` -- calls BrowserBase/Stagehand to take PNG screenshot, uploads to S3, returns URL
- Store screenshots in `ghlWebhookEvents` or a new `executionScreenshots` array on the execution record
- Display captured screenshots in a slide-out gallery (reuse `ScreenshotGallery.tsx`)

### 4. Session Recording Link

After execution completes:
- BrowserBase provides a recording URL for each session
- Store the recording URL on the execution record
- Show "View Recording" button in the preview panel and in execution history
- Link to BrowserBase replay page (opens in new tab)

### 5. Connection Resilience

Enhance iframe reconnection:
- If iframe loses connection (session timeout), show reconnection overlay
- Auto-refresh iframe every 30s if SSE is disconnected (fallback polling already exists in AgentDashboard at 5s interval)
- Show clear "Session Ended" state when execution completes with link to recording

### 6. Mobile Improvements

Existing mobile Sheet for preview is functional -- enhance:
- Add zoom controls to mobile view
- Add screenshot button to mobile view
- Show action overlay in mobile view

## Verification

1. Start an agent task -> live preview iframe loads in right panel
2. Status bar shows "Live" indicator with green dot
3. Click zoom 75% -> iframe scales down, showing more of the page
4. Click "Capture Screenshot" -> screenshot saved, viewable in gallery
5. Action overlay shows current agent action in real-time (e.g., "Navigating to...")
6. Execution completes -> "View Recording" button appears with BrowserBase link
7. Pause button pauses agent -> resume continues
8. SSE disconnection -> status bar shows "Polling" mode, preview still updates
9. Mobile: preview sheet has zoom controls and screenshot button

## Risks

- BrowserBase debug URL iframes may have CORS restrictions with zoom transforms
- Screenshot capture via BrowserBase API may have latency (1-3s)
- Session recording URLs may not be immediately available after session ends
- Zoom transform can cause iframe interaction issues (clicks offset); may need to disable zoom during interaction
