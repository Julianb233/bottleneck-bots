# Phase 03-02: GHL Contact Management Functions

**Day**: 3 (Wed 3/18)
**Scope**: Contact CRUD, bulk operations, tagging, custom fields
**PRD Coverage**: FR-007 through FR-014 (Contact Management)
**Depends on**: 03-01 (GHL OAuth + Service Layer)

---

## Goal

Implement the full GHL contact management API surface as methods on `GHLService` and expose them as tRPC procedures. This covers 8 functional requirements: CRUD, bulk import/export, search, tags, custom fields, activity history, and contact merging.

## Tasks

### 1. Implement Contact Methods on GHLService

Add a `contacts` namespace to `ghl.service.ts`:

```typescript
// GHL API v2 base: https://services.leadconnectorhq.com
// All endpoints require locationId in headers as "Version: 2021-07-28"

contacts: {
  // FR-007: Contact CRUD
  create(userId, locationId, data: GHLContactCreate): Promise<GHLContact>
    // POST /contacts/ with body { firstName, lastName, email, phone, ... }

  get(userId, locationId, contactId: string): Promise<GHLContact>
    // GET /contacts/{contactId}

  update(userId, locationId, contactId: string, data: Partial<GHLContactCreate>): Promise<GHLContact>
    // PUT /contacts/{contactId}

  delete(userId, locationId, contactId: string): Promise<void>
    // DELETE /contacts/{contactId}

  // FR-010: Contact Search
  search(userId, locationId, query: GHLContactSearch): Promise<GHLContactSearchResult>
    // GET /contacts/ with query params: query, limit, startAfter, startAfterId
    // Support filters: email, phone, tags, dateAdded range, customField values

  // FR-008: Bulk Import (up to 50K records)
  bulkImport(userId, locationId, contacts: GHLContactCreate[]): Promise<GHLBulkImportResult>
    // Chunked processing: split into batches of 100
    // POST /contacts/ for each (GHL has no native bulk endpoint)
    // Track progress, return { imported, failed, duplicates, errors[] }
    // Use BullMQ job queue for large imports (>500 contacts)

  // FR-009: Bulk Export
  bulkExport(userId, locationId, filters?: GHLContactSearch): Promise<GHLContact[]>
    // Paginate through GET /contacts/ collecting all results
    // Return full contact array (caller handles CSV conversion)

  // FR-011: Tag Management
  addTag(userId, locationId, contactId: string, tags: string[]): Promise<void>
    // POST /contacts/{contactId}/tags with { tags }

  removeTag(userId, locationId, contactId: string, tags: string[]): Promise<void>
    // DELETE /contacts/{contactId}/tags with { tags }

  listTags(userId, locationId): Promise<string[]>
    // GET /locations/{locationId}/tags

  // FR-012: Custom Fields
  getCustomFields(userId, locationId): Promise<GHLCustomField[]>
    // GET /locations/{locationId}/customFields

  updateCustomField(userId, locationId, contactId: string, fieldKey: string, value: unknown): Promise<void>
    // PUT /contacts/{contactId} with { customField: { [fieldKey]: value } }

  // FR-013: Activity History
  getActivity(userId, locationId, contactId: string): Promise<GHLActivity[]>
    // GET /contacts/{contactId}/tasks (notes/tasks/activities)

  // FR-014: Contact Merge
  merge(userId, locationId, primaryId: string, duplicateIds: string[]): Promise<GHLContact>
    // POST /contacts/{primaryId}/merge with { duplicateContactIds }
}
```

### 2. Define GHL Contact Types

**File**: `server/services/ghl.types.ts`

```typescript
interface GHLContactCreate {
  firstName?: string;
  lastName?: string;
  name?: string;
  email?: string;
  phone?: string;
  address1?: string;
  city?: string;
  state?: string;
  postalCode?: string;
  website?: string;
  timezone?: string;
  dnd?: boolean;
  tags?: string[];
  source?: string;
  customField?: Record<string, unknown>;
}

interface GHLContact extends GHLContactCreate {
  id: string;
  locationId: string;
  dateAdded: string;
  dateUpdated: string;
  attachments?: string[];
  assignedTo?: string;
}

interface GHLContactSearch {
  query?: string;           // Free text search
  email?: string;
  phone?: string;
  tags?: string[];
  startAfter?: number;      // Pagination cursor
  startAfterId?: string;
  limit?: number;           // Max 100
  dateAddedFrom?: string;
  dateAddedTo?: string;
}

interface GHLContactSearchResult {
  contacts: GHLContact[];
  meta: { total: number; startAfter: number; startAfterId: string; };
}

interface GHLBulkImportResult {
  total: number;
  imported: number;
  failed: number;
  duplicates: number;
  errors: Array<{ index: number; error: string; contact: Partial<GHLContactCreate> }>;
}

interface GHLCustomField {
  id: string;
  name: string;
  fieldKey: string;
  dataType: string;
  position: number;
}

interface GHLActivity {
  id: string;
  contactId: string;
  type: string;
  body: string;
  createdAt: string;
}
```


Zod validation schemas for each input/output.

### 4. Bulk Import Worker

**File**: `server/workers/ghl-bulk-import.worker.ts`

For imports > 500 contacts:
- Accept job from BullMQ queue `ghl-bulk-import`
- Process in batches of 100 with rate limiting
- Track progress in `ghlSyncLog` table
- Emit SSE progress events via `AgentSSEEmitter`
- Handle duplicates (GHL returns 409 for duplicate email/phone)
- Return final `GHLBulkImportResult`

### 5. Contact Sync Tracking

Add to `drizzle/schema-ghl.ts`:
- `ghlContactSync` table: userId, locationId, contactId, ghlContactId, lastSyncAt, syncDirection, status

This enables future bidirectional sync (Phase 2 feature).

2. Search contacts by email -> returns matching results
3. Add tags to contact -> tags visible in GHL
4. Update custom field -> field value updated in GHL
5. Bulk import 100+ contacts -> all created, progress tracked, duplicates reported

7. Get activity -> returns timeline entries
8. Merge contacts -> duplicates merged into primary
9. All operations logged in `ghlSyncLog`

## Risks

- GHL has no native bulk import endpoint; must loop POST /contacts (rate limit sensitive)
- Contact search is cursor-based, not offset-based; pagination needs startAfterId
- Custom field keys vary per location; must fetch fields first to validate
- Merge endpoint may not be available on all GHL plans
