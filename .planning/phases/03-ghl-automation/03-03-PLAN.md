**Day**: 3 (Wed 3/18)

**PRD Coverage**: FR-015 through FR-019, FR-031 through FR-039, FR-044 through FR-047

## Goal

Implement pipeline automation, campaign management, workflow triggers, and communication functions on the GHL service. This covers 20+ functional requirements spanning opportunities, campaigns, workflows, and messaging.

## Tasks

### 1. Pipeline & Opportunity Methods

Add to `ghl.service.ts` under `pipelines` and `opportunities` namespaces:

```typescript
pipelines: {
  // FR-017: Pipeline Listing
  list(userId, locationId): Promise<GHLPipeline[]>
    // GET /opportunities/pipelines

  getStages(userId, locationId, pipelineId: string): Promise<GHLPipelineStage[]>
    // Included in pipeline list response
}

opportunities: {
  // FR-015: Opportunity CRUD
  create(userId, locationId, data: GHLOpportunityCreate): Promise<GHLOpportunity>
    // POST /opportunities/

  get(userId, locationId, opportunityId: string): Promise<GHLOpportunity>
    // GET /opportunities/{opportunityId}

  update(userId, locationId, opportunityId: string, data: Partial<GHLOpportunityCreate>): Promise<GHLOpportunity>
    // PUT /opportunities/{opportunityId}

  delete(userId, locationId, opportunityId: string): Promise<void>
    // DELETE /opportunities/{opportunityId}

  // FR-016: Stage Management
  moveStage(userId, locationId, opportunityId: string, stageId: string): Promise<GHLOpportunity>
    // PUT /opportunities/{opportunityId} with { pipelineStageId: stageId }

  // FR-018: Assignment
  assign(userId, locationId, opportunityId: string, assignedTo: string): Promise<GHLOpportunity>
    // PUT /opportunities/{opportunityId} with { assignedTo }

  // FR-019: Value Tracking
  updateValue(userId, locationId, opportunityId: string, monetaryValue: number): Promise<GHLOpportunity>
    // PUT /opportunities/{opportunityId} with { monetaryValue }

  // Search/list with filters
  search(userId, locationId, filters: GHLOpportunitySearch): Promise<GHLOpportunitySearchResult>
    // GET /opportunities/search with query params
}
```

### 2. Campaign Methods

Add `campaigns` namespace:

```typescript
campaigns: {
  // FR-031-035: Campaign Operations
  list(userId, locationId): Promise<GHLCampaign[]>
    // GET /campaigns/

  // FR-035: Create Campaign
  create(userId, locationId, data: GHLCampaignCreate): Promise<GHLCampaign>
    // POST /campaigns/ (if GHL API supports; otherwise note limitation)

  // FR-031: Add Contact to Campaign
  addContact(userId, locationId, campaignId: string, contactId: string): Promise<void>
    // POST /contacts/{contactId}/campaigns/{campaignId}

  // FR-032: Remove Contact from Campaign
  removeContact(userId, locationId, campaignId: string, contactId: string): Promise<void>
    // DELETE /contacts/{contactId}/campaigns/{campaignId}

  // FR-033: Pause Campaign for Contact
  pauseForContact(userId, locationId, campaignId: string, contactId: string): Promise<void>
    // POST /contacts/{contactId}/campaigns/{campaignId}/pause (or equivalent)

  // FR-034: Campaign Stats
  getStats(userId, locationId, campaignId: string): Promise<GHLCampaignStats>
    // GET /campaigns/{campaignId}/stats (or aggregated from contact data)

  // Bulk add contacts to campaign
  bulkAddContacts(userId, locationId, campaignId: string, contactIds: string[]): Promise<{ added: number; failed: number }>
    // Loop POST for each contact, respecting rate limits
}
```

## Verification

1. List pipelines -> returns pipeline names and stages from GHL
2. Create opportunity -> appears in GHL pipeline
3. Move opportunity to new stage -> stage updated in GHL
4. List campaigns -> returns campaign list
5. Add contact to campaign -> contact enrolled in GHL
6. Trigger GHL workflow -> workflow executes in GHL
7. Send SMS via `messages.sendSMS` -> SMS received by contact
8. GHL nodes appear in workflow builder palette under "GoHighLevel" category

## Risks

- Some GHL API v2 endpoints are location-scoped and require correct `Version` header
- Campaign creation may not be supported via API (GHL API limitations vary by plan)
- Workflow trigger API may have limited data passing capabilities
- SMS/email sending requires the GHL location to have configured messaging providers
