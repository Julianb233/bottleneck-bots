## Tasks

### 1. GHL Sandbox Setup

- Use an existing GHL developer account or create a sandbox location
- Configure environment variables: `GHL_CLIENT_ID`, `GHL_CLIENT_SECRET`, `GHL_REDIRECT_URI`, `GHL_TEST_LOCATION_ID`, `GHL_TEST_ACCESS_TOKEN`
- For integration tests, use a pre-authorized access token (skip OAuth flow in tests)
- Create test data helpers for generating realistic contact/opportunity data

### 2. Test Suite: OAuth Flow (`tests/integration/ghl/oauth.test.ts`)

```
- initiateAuthorization returns valid URL with correct params (state, code_challenge, scopes)
- handleCallback exchanges code for tokens (mock GHL token endpoint)
- Token storage encrypts and stores via credential vault
- Token refresh triggers before expiry
- Token revocation removes credentials
- Multi-location: same user can connect multiple locations
- Invalid state parameter rejects callback
```

### 3. Test Suite: Contacts (`tests/integration/ghl/contacts.test.ts`)

```
- create contact with required fields -> returns contact with ID
- create contact with all optional fields -> returns complete contact
- get contact by ID -> matches created contact
- update contact fields -> fields updated
- delete contact -> contact no longer retrievable
- search by email -> finds contact
- search by phone -> finds contact
- search by tag -> finds tagged contacts
- search with date range -> filters correctly
- addTag -> tag appears on contact
- removeTag -> tag removed from contact
- listTags -> returns all location tags
- getCustomFields -> returns location custom fields
- updateCustomField -> value persisted
- getActivity -> returns activity entries
- merge contacts -> primary has merged data, duplicates removed
- bulkImport 10 contacts -> all created, result counts correct
- bulkImport with duplicates -> duplicate count accurate
- bulkExport -> returns all contacts
```

### 7. Test Suite: Appointments (`tests/integration/ghl/appointments.test.ts`)

```
- list calendars -> returns calendars

- get appointment -> matches created
- update appointment -> fields updated

```

### 8. Test Suite: Webhooks (`tests/integration/ghl/webhooks.test.ts`)

```
- valid HMAC signature -> event accepted (200)
- invalid HMAC signature -> rejected (401)
- duplicate eventId -> accepted but not reprocessed (idempotent)
- contact.created event -> handler processes, status=processed
- opportunity.stage.changed event -> handler triggers, status=processed
- form.submitted event -> contact created/updated
- handler failure -> event retried with backoff
- 5 failures -> event moved to dead letter queue
- retry dead letter event -> reprocessed
```

### 9. Test Suite: Rate Limiting (`tests/integration/ghl/rate-limiting.test.ts`)

```
- 100 rapid requests -> all succeed (within limit)
- 150 rapid requests -> requests 101+ queued and retried
- 429 response -> automatic retry with backoff
- Circuit breaker opens after 5 consecutive failures -> fast-fail
- Circuit breaker half-opens after 30s -> test request goes through
```

### 10. Test Suite: Multi-Tenant Isolation (`tests/integration/ghl/isolation.test.ts`)

```
- User A cannot access User B's GHL connections
- User A cannot call GHL API for User B's locationId
- All operations logged in audit log
- Permission check enforces ghl_read/ghl_write permissions
```

### 11. Test Data Cleanup Script

**File**: `scripts/ghl-test-cleanup.ts`

- Delete all test contacts (tagged with `_test_`)
- Delete all test opportunities (named with `[TEST]` prefix)
- Delete all test appointments
- Clear test webhook events from DB
- Idempotent -- safe to run multiple times

### 12. GHL API Quirks Documentation

**File**: `docs/ghl-api-quirks.md`

Document during testing:
- Rate limit behavior (exact headers, reset timing)
- Endpoint differences between v1 and v2
- Missing or undocumented endpoints
- Pagination behavior (cursor vs offset)
- Required vs optional fields (what GHL actually enforces vs docs)
- Error response formats
- Webhook payload structure per event type
- Any plan-based limitations encountered
