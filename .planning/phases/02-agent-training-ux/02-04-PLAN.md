# Phase 02 Plan 4: RAG-to-Execution Pipeline

## Goal
Agents reference uploaded training documents during task execution, completing the loop from document upload to agent-informed actions.

## Prerequisites
- RAG system operational:
  - `server/services/rag.service.ts` with `RAGService` class (ingest, retrieve, buildContext, buildSystemPrompt)
  - `drizzle/schema-rag.ts` defines `documentationSources`, `documentationChunks`, `platformKeywords`
  - Embeddings via `server/rag/embeddings.ts` (OpenAI embeddings, pgvector similarity search)
  - `server/api/routers/rag.ts` with upload/ingest/list/delete endpoints
- SOP system exists:
  - `drizzle/schema-sop.ts` defines `sopDocuments`, `sopSteps`, `sopExecutions` tables
  - `server/api/routers/sop.ts` with full SOP CRUD
- Agent orchestrator exists in `server/api/routers/agent.ts`
- Document upload working in Training.tsx Documents tab

## Tasks

1. **Verify existing RAG pipeline works end-to-end**
   - Test: upload a PDF via Training.tsx -> verify it appears in `documentation_sources` table
   - Test: verify chunks are created in `documentation_chunks` with embeddings
   - Test: call `ragService.retrieve("some query")` and confirm relevant chunks are returned
   - Test: `ragService.buildSystemPrompt("task description")` returns enriched prompt with context
   - Fix any broken links in the chain (embedding generation, vector search, content retrieval)

2. **Add SOP-specific document processing**
   - When a document is uploaded with category "sop", apply special processing:
     a. Extract numbered steps from the document content (regex + LLM-assisted)
     b. Tag each chunk with step number metadata for ordered retrieval
     c. Assign priority ranking to chunks (steps > context > appendix)
   - Create `server/services/sopProcessor.ts`:
     - `processSopDocument(content: string)`: returns structured steps
     - `tagChunksWithStepInfo(chunks, steps)`: adds step metadata to chunk records
   - Update `rag.service.ts` `ingest()` to check category and call SOP processor when appropriate
   - Store SOP metadata in chunk's `metadata` jsonb field: `{ sopStep: 3, priority: "high", actionType: "browser" }`

3. **Build knowledge base browser UI**
   - New section in Documents tab or standalone component
   - List all uploaded documents with:
     - Title, category, platform, chunk count, upload date
     - Expand to preview individual chunks with their metadata
     - Similarity search: enter a query, see which chunks would be retrieved
   - Delete document (cascades to chunks): already working via `trpc.rag.deleteSource`
   - Re-process button: delete chunks, re-chunk and re-embed (uses `ragService.updateSource`)
   - Filter by category (training, sop, business, technical, general) and platform

4. **Connect RAG to agent orchestrator**
   - Before agent starts executing a task, call `ragService.buildSystemPrompt(taskDescription)`
   - Inject retrieved documentation context into the agent's system prompt
   - Pass detected platforms to guide tool selection (e.g., if GHL docs retrieved, prefer GHL tools)
   - In `server/api/routers/agent.ts` (or the agent service it calls):
     a. At task start: `const ragResult = await ragService.buildSystemPrompt(task.description)`
     b. Prepend `ragResult.systemPrompt` to the agent's system message
     c. Log `ragResult.retrievedChunks` for observability
   - If behavior config has custom instructions, append those to the system prompt too

5. **Add RAG context to agent thinking steps**
   - When agent references a training doc during execution, note it in the thinking steps
   - Format: "Referenced document: [title] (chunk X, relevance: Y%)"
   - Store in `agentExecutions.thinkingSteps` jsonb array
   - Display in agent execution UI so users can see which docs influenced the agent

6. **Add feedback loop for RAG quality**
   - After task completion, check if retrieved docs were actually relevant
   - Track: were retrieved chunks referenced in the final output?
   - Store relevance feedback in chunk metadata to improve future retrieval
   - Optionally: user can rate "was this document helpful?" after seeing agent results
